## Project Notes

MoSAIC-P38 Jekyll site (jekyll-theme-yat) — architecture map & gotchas after a maintainability refactor.

STRUCTURE / WHO OWNS WHAT:
- Theme: jekyll-theme-yat 1.10.0 (gem). Project files override gem files by same path under _includes/, _layouts/, _sass/.
- assets/css/main.scss is the SCSS entry point. Order is load-bearing: set $theme-color/$brand-color/$content-width BEFORE `@import "yat"`, then `@import "custom/lbnl"` LAST so brand overrides win.
- _sass/custom/_lbnl.scss is the single source of truth for brand colors (CSS vars --lbnl-*), Libre Franklin font, and the 3 custom regions: .lbnl-logo-bar, .site-header (teal nav), .lbnl-hero.
- Custom includes: _includes/logo-bar.html, _includes/hero.html, and _includes/head.html (overrides theme head — only adds Libre Franklin font; MUST keep main.css link + <head> wrapper or whole site goes unstyled).
- Layouts: _layouts/home.html is a STANDALONE copy of the theme default.html with the banner include OMITTED (that's how we kill "Your awesome heading"). _layouts/page.html wraps default (theme has no page layout).

KEY GOTCHAS:
1. yat's default.html ALWAYS includes views/banner.html. The banner pulls heading/banner from _data/defaults.yml (home.heading="Your awesome heading", home.banner="Your image url"). To remove it on the home page we reimplement default.html sans banner in home.html. Front-matter `banner: false` does NOT work — Liquid turns it into truthy `/false` via relative_url, so has_banner stays true.
2. index.md needs `header_transparent: false`. Otherwise views/header.html computes has_banner from the home.banner default and adds site-header-transparent, which at data-scroll-status=top sets header height:0 + transparent bg, hiding the teal nav.
3. $content-width is referenced by yat _base.scss line 186 but left undefined → Sass compile crash. Must define in main.scss.
4. Dead selectors removed from _lbnl.scss: .navbar, .header, .post-top-bar, nav.navbar, .nav-link, .navbar .menu a, .navbar-nav .nav-item a. yat emits NONE of these — it only uses .site-header / .site-header-inner / .site-brand / .site-nav / .page-link. Verified by grepping all built pages: zero matches. Safe to remove.
5. Two image dirs existed historically: top-level images/ had the REAL assets; assets/images/ had 0-byte placeholders. HTML references /assets/images/. Real ones were copied in. Keep assets/images/ as the live path.

DEPLOY: classic github-pages gem rejects yat (not on theme whitelist). Deploy via .github/workflows/deploy.yml (Actions, actions/deploy-pages). Requires repo Settings>Pages>Source=GitHub Actions.

VERIFICATION TECHNIQUE THAT WORKED: snapshot _site to /tmp before refactor, rebuild, `diff -rq`, then for each differing file diff with comments/blank lines filtered (grep -vE '^\s*(//|/\*|\*|#|$)') to prove only comments changed. For CSS the only real delta was the dead-selector removal; confirmed those classes appear nowhere in built HTML. This is the right pattern for "refactor without changing appearance" tasks.
